A Tour of C++

written by yannic on 2016-02-17

Acknowledgement

This blog post is a review of the book "A Tour of C++" (Bjarne Stroustrup, Addison-Wesley. ISBN 978-0321958310. September 2013.).

Motivation

I recently finished reading "A Tour of C++" by Bjarne Stroustrup. It is a 180 page book describing literally all of modern C++ in a clear and concise way.

In my opinion, the book is a great way to get (re-)familiarized with the C++ programming language in general and new language additions in particular.

It is structured in 14 chapters covering topics such as core language features (i.e. classes and templates) as well as standard library features (i.e. stream, containers, algorithms).

Each point is illustrated with descriptions, explanations and code samples. The ending of each chapter is marked by general advice about the content of the chapter.

Key takeaways

  • Use braced initializton for declarations with a named type, assignment for initializing declarations with auto
int x {42};
auto y = 4711;
  • Return containers by value (relying on move semantics)
Vector f()
{
    Vector x(1000);
    Vector y(1000);
    Vector z(1000);
    z = x; // we get a copy
    y = std::move(x); // we get a move
    return z; // we get a move
};
  • Use variadic templates to define a function that takes a variable number of arguments of a variety of types
void f() { } // do nothing

template<typename T>
void g(T x) { cout << x << " "; }

template<typename T, typename ... Tail>
void f(T head, Tail... tail)
{
    g(head); // do something to head
    f(tail...); // try again with tail
}

f(1,2.2,"hello"); // prints 1 2.2 hello
  • For simple traversals of a container, use a range-for loop or a begin/end pair of iterators
void print_book(const vector<Entry>& book, const vector<int>& nums)
{
    for (const auto& x : book)
        cout << x << '\n';

    for (auto n : nums)
        cout << n << '\n';
}

C++ Type Deduction

written by yannic on 2016-02-10

Acknowledgement

This blog post is a summary of the talk "Type Deduction and Why You Care" (Scott Meyers, CppCon 2014).

You can find the talk here and the slides here. You can find more of Scott's talks here.

Please refer to the actual talk for in depth information and code samples.

Scope of C++ Type Deduction

  • C++98: Templates
  • C++11: auto variables, universal references, lambda captures and returns, decltype
  • C++14: Function return types, lambda init captures.

Templates

Template type deduction can be generalised by three cases:

  • ParamType is a reference or pointer, but not a universal reference
    • T will not be deduced to a reference
  • ParamType is a universal reference
    • T will be deduced to a reference if expression is an lvalue (e.g. if you can take its address)
  • ParamType is neither reference nor pointer
    • (Top level) reference-/const-qualifiers will be dropped when deducing T

Auto

The rules of template type deduction also apply to auto, with one caveat.

auto will be deduced to a std::initializer_list when initialized with or assigned a braced initializer list:

Careful: This behaviour will change with the adoption of proposal N3922 to C++17 (already implemented in MSVC15 CTP).

Lambda captures

Lambda capture type deduction uses three different mechanisms:

  • By reference captures use template type deduction (for reference params).
  • Lambda init captures use auto type deduction.
  • By value captures use template type deduction with const/volative qualifier retention

decltype

The decltype keyword deduces the declared type of a name or an lvalue expression with const/volatile/reference retention.

decltype(expr) typically deduces a reference type. The decltype(name) rule beats the decltype(lvalue expression) rule.

Function return types

In C++11, function return type deduction applies to single statement lambdas only.

In C++14, the support for function return type deduction extends to all lambdas and all functions.

The following return type specifiers are legal:

  • auto
    • Uses template rules
  • decltype(auto)
    • Uses decltype rules

Either can be correct, it depends on the use case. Rule of thumb:

  • Use auto if a reference type would never be correct.
  • Use decltype(auto) only if a reference type could be correct.
